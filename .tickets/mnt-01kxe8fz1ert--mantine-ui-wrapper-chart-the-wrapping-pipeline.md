## Destination

A locked design + de-risked wrapping pipeline for `mantine-ui-wrapper`: a framework-agnostic ClojureScript wrapper of Mantine (pinned to Mantine 9.4.1), with rich per-component factories generated from Mantine's `docgen.json`. Proven by a PoC that builds and renders one of each pattern — a few codegen'd core components (incl. polymorphic `component=`, `styles`/`classNames`, section props), one imperative API (e.g. `notifications.show`), and one hook (e.g. `useDisclosure`) — in a shadow-cljs harness with Mantine's CSS actually loading. When that renders, remaining work is mechanical fan-out across packages.

## Notes

- Inspiration: fulcrologic/semantic-ui-wrapper (same idea, for Mantine).
- Framework-agnostic CLJS (usable from Fulcro, Reagent/re-frame, UIx, Helix, raw interop).
- PINNED VERSION: Mantine 9.4.1 (all @mantine/* packages aligned; docgen tool mantine-docgen-script@^1.6.0).
- Rich factories: per-component metadata from Mantine's `docgen.json` — generated by `tsx scripts/docgen` into `apps/mantine.dev/src/.docgen/docgen.json` (gitignored; must be generated).
- The Mantine source is NOT checked out in this repo; research tickets clone it.
- Three wrapping tracks: components (codegen) / imperative APIs / hooks.
- EXECUTION OVERRIDE: unlike a pure planning map, the terminal ticket carries execution — it builds a rendering PoC to de-risk the pipeline. Everything before it is decisions.
- Skills to consult per session: /grilling, /domain-modeling, /prototype, /research.

## Decisions so far

- Research: how semantic-ui-wrapper's codegen works ([docs/research/reference-codegen.md](docs/research/reference-codegen.md)) — reads pre-built JSON (analogue of docgen.json) via a plain-Clojure generator; emits one .cljc per component + aggregate; factory = def-bound fn; its prop conversion is bare clj->js (NO kebab->camel / :style / events) — the deliberate divergence point for us.
- Research: Mantine docgen.json ([docs/research/mantine-docgen.md](docs/research/mantine-docgen.md)) — object keyed by component name -> {props:{name,description,required,defaultValue,type:{name}}}; ~250 entries incl. core/dates/charts; but STRIPS polymorphic/style-system props (component/className/styles/variant/spacing) + component descriptions; NO hooks, NO imperative APIs. Pin Mantine 9.4.1.
- Research: Mantine CSS + npm-interop ([docs/research/mantine-css-interop.md](docs/research/mantine-css-interop.md)) — pre-compiled plain CSS per package (@mantine/<pkg>/styles.css, core first; layered variant available; modals ships none); PostCSS NOT needed by consumers; shadow-cljs does NOT process CSS imports — deliver via <link> from node_modules in the harness; wrapper itself imports no CSS.
- Research: imperative + hooks API surface ([docs/research/imperative-and-hooks-api.md](docs/research/imperative-and-hooks-api.md)) — 3 imperative packages share one shape (callable singleton + a renderer mounted once); 79 hooks enumerable from package barrel, with a load-bearing tuple-vs-object return split.
- [Decide: props-conversion semantics](mnt-01kxe8gzn6bt) — ONE public converter fn: hybrid kebab->camel keys (data-/aria- exempt, round-trip-safe); shallow-default values + curated deep-convert set {style,styles,classNames,vars} (map->convert, fn->passthrough); style leaf (--* verbatim, numeric passthrough) + class leaf (collection space-join); :class alias+merge; children path (prune nil / flatten collections / render-prop passthrough, :key rides in child props); component/renderRoot/section props pure passthrough; raw JS SyntheticEvent to handlers; reserved escape-hatch merge key (plain clj->js, wins last). Deliberate divergence from SUIW's bare clj->js. Controlled-input wrapping + factory call convention pushed to the codegen ticket.
- [Decide: codegen pipeline & generated output](mnt-01kxe8gzrhy6) — Generator = headless/CI-runnable/pinned plain-Clojure transform of a COMMITTED docgen.json + installed node_modules -> ONE .cljc ns per package (mantine.core/dates/charts, NO aggregate). Component defs: kebab-case, no prefix, per-component named-export requires (tree-shakeable under :advanced); within-pkg kebab collisions HARD-ERROR; auto (:refer-clojure :exclude); package + require-string resolved from node_modules exports (unresolvable logged+skipped). Docstring-ONLY at runtime (metadata stays generator-side); docstring = docgen prop-list + best-effort mantine.dev URL. Thin generated files delegate to hand-written mantine.impl.props (converter) + mantine.impl.factory (variadic-def factory + controlled-input React shim on a curated set + not-implemented). Generated nses .cljc & JVM-loadable — :cljs = real factory, :clj = call-time not-implemented throw. Optional props via map? first-arg detection. Handoffs to build-tooling: concrete react/createElement require form + generator runner mechanism (lean bb).
- [Decide: hooks wrapping pattern](mnt-01kxe8h01n5p) — GENERATED mantine.hooks .cljc ns, use* hooks ONLY (~79; non-hook barrel utils excluded), inheriting all component-codegen conventions. Returns: ZERO conversion / raw passthrough (tuples destructure positionally off the raw JS array, objects via interop, scalars as-is — NO shape table; matches near-unanimous CLJS ecosystem norm: UIx/Helix/Reagent/rum/Fulcro). Args: raw JS-shaped (#js ...), public props converter available OPT-IN — no per-hook arg metadata. Generated form = thin def-alias w/ docstring: (def use-x "<doc>" #?(:cljs <import> :clj <not-implemented throw>)) — no defn/factory. Docstrings RICH: description extracted from apps/mantine.dev/src/mdx/data/mdx-hooks-data.ts (regex hDocs(name,desc) + useElementSize inline exception) + derivable mantine.dev/hooks/use-* URL. Rules-of-hooks NOT enforced (delegated to React); ^js/js-interop needed on object returns under :advanced. Handoffs to build-tooling: (a) NEW committed generator input = extracted {hook->description} (mdx source lives in docs-app, NOT node_modules); (b) barrel use* enumeration + per-symbol require resolution.
- Research: component MDX metadata ([docs/research/component-mdx-metadata.md](docs/research/component-mdx-metadata.md)) — CONFIRMS the docs-app carries per-component prose the way hooks do, fillable into component docstrings (REFINES the codegen docstring spec mnt-01kxe8gzrhy6, which was docgen prop-list + URL only). Source: apps/mantine.dev/src/mdx/data/mdx-{core,dates,charts}-data.ts — inline object literals keyed by EXACT PascalCase component name (matches docgen.json keys), ~140+ entries each w/ a one-line `description` + `polymorphic` flag + `props[]`/`styles[]` key-sets + `componentPrefix`. Extraction LOW difficulty (Mantine's own scripts/llm/compile-mcp-data.ts regex-parses these; all descriptions plain single-quoted single-line). Adopt = mirror hooks: commit {component->{description,polymorphic,...}} input merged into docstrings. Caveats: compound sub-components (Button.Group, Table.Tr) are NOT top-level keys (they live in the parent's props[]/styles[]) → inherit parent desc or stay prose-less; category/group unset; the real Styles API selectors/CSS-vars are a separate source (see Not yet specified).

## Not yet specified

- The mechanical fan-out itself: full coverage of every core component + full dates/charts.
- Full imperative-API coverage (all notifications/modals/spotlight fns).
- Full hooks coverage (all 79).
- Generating specs/malli (or richer validation) from extracted prop metadata.
- Testing / CI strategy for generated wrappers.
- Mantine version-bump strategy (re-running codegen against a newer major).
- Wrapping the non-hook @mantine/hooks barrel utilities (randomId, getHotkeyHandler, readLocalStorageValue/readSessionStorageValue, mergeRefs, mask helpers, ...) — excluded from the hooks pattern (no docstring source; `range` collides with clojure.core). Revisit if a specific util is needed (e.g. randomId for imperative-API ids).
- Extracting the real Mantine Styles API selectors / CSS variables (per-component selector names + --var lists) from packages/@docs/styles-api/src/data/*.styles-api.ts — a separate, backtick-heavy source needing a real parse; a further docstring/metadata enrichment beyond the easy prose-description work confirmed above.

## Out of scope

- Clojars publishing, release process, versioning.
- Supporting multiple Mantine majors simultaneously.
- Non-official / community Mantine extensions.
